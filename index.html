1. DNS Fundamentals

What is DNS (Domain Name System)?

DNS, or Domain Name System, is like the phone book of the internet. Just as you look up a person's name to find their phone number, DNS translates human-readable domain names (like example.com) into IP addresses (like 93.184.216.34) that computers use to communicate with each other.

Why do we need DNS?
  • Humans are better at remembering names like google.com than numbers like 142.250.191.14
  • IP addresses can change, but domain names remain constant
  • One domain can point to multiple IP addresses for load balancing and redundancy
  • Makes the internet more user-friendly and manageable

2. DNS Resolution Process - Step by Step

What Happens When You Type example.com?

Let's trace through the complete DNS resolution process with animations:

1 Browser Cache Check

Browser checks if it has the IP address cached. If found, uses cached value (fastest).

2 OS Cache Check

Operating system checks its DNS cache. Windows: ipconfig /displaydns, Linux: system cache.

3 Recursive Resolver Query

If not in cache, query goes to recursive resolver (usually ISP's DNS or 8.8.8.8). Resolver performs iterative queries.

4 Root Nameserver Query

Resolver asks root nameserver: "Where is .com?" Root responds: "Ask these TLD nameservers for .com"

5 TLD Nameserver Query

Resolver asks TLD nameserver: "Where is example.com?" TLD responds: "Ask these authoritative nameservers"

6 Authoritative Nameserver Query

Resolver asks authoritative nameserver: "What is the IP address of example.com?" Gets A record: "93.184.216.34"

7 Response Caching

Resolver caches the response. Returns IP address to your computer. Your computer caches it too.

8 HTTP/HTTPS Request

Browser makes HTTP/HTTPS request to the IP address. Website loads!

4. Hostinger vs Route 53: Complete Comparison

Feature Hostinger DNS AWS Route 53
Cost Free with domain registration $0.50/hosted zone/month + $0.40 per million queries
Ease of Use Very user-friendly, simple interface More complex, requires AWS knowledge
Performance Good, global CDN Excellent, global network of DNS servers
Availability High (99.9%+) Very High (99.99% SLA)
Advanced Features Basic DNS features Advanced routing, health checks, failover
AWS Integration None Deep integration with all AWS services
Geographic Routing No Yes (latency-based, geolocation)
Health Checks No Yes
Failover No Yes (automatic)

When to Choose Hostinger DNS

  • Simple websites and blogs
  • Cost is a primary concern
  • You want simplicity
  • Static websites
  • You're already using Hostinger hosting

When to Choose Route 53

  • You're using AWS services (EC2, Load Balancer, CloudFront)
  • You need high availability (99.99% uptime)
  • You need advanced routing (geographic, latency-based)
  • You need health checks and automatic failover
  • You need scalability for high-traffic websites

6. Advanced Route 53 Features

Routing Policies

‚öñÔ∏è

Weighted Routing

Route traffic to multiple resources with weights. Higher weight = more traffic. Great for A/B testing and gradual migration.

‚ö°

Latency-Based Routing

Routes traffic to the resource with the lowest latency from the user's location. Perfect for global applications.

üîÑ

Failover Routing

Routes traffic to primary resource. If primary fails health check, automatically routes to secondary. High availability!

üåç

Geolocation Routing

Routes traffic based on user's geographic location. Great for content localization and compliance requirements.

Health Checks

What are Health Checks? Monitor the health of your resources and automatically route traffic away from unhealthy resources.

Types:

  • Endpoint Health Checks: Monitor HTTP/HTTPS endpoints
  • CloudWatch Alarms: Monitor CloudWatch metrics
  • Calculated Health Checks: Combine multiple health checks

Alias Records

Alias vs CNAME

7. DNS Record Types - Complete Reference

A Record (Address Record)

Purpose: Maps a domain name to an IPv4 address

When to use: Pointing domain to server's IP address, root domain, subdomains

Example A Record 
# Root domain
example.com    A    93.184.216.34    TTL: 3600

# Subdomain
api.example.com    A    192.168.1.100    TTL: 3600

# Multiple A records (load balancing)
example.com    A    93.184.216.34
example.com    A    93.184.216.35
example.com    A    93.184.216.36

AAAA Record (IPv6 Address)

Example AAAA Record 
example.com    AAAA    2606:2800:220:1:248:1893:25c8:1946    TTL: 3600

CNAME Record (Canonical Name)

Important Rules:
  • Cannot use CNAME on root domain if you have other records (MX, NS, etc.)
  • CNAME cannot coexist with other record types for the same name
  • CNAME target must exist and resolve to an IP
Example CNAME Record

# CDN integration

8. Load Balancer Integration with Route 53

Why Use Load Balancer?

  • Distribute traffic across multiple EC2 instances
  • High availability (if one instance fails, others handle traffic)
  • SSL termination at load balancer
  • Health checks at load balancer level
  • Single endpoint (load balancer DNS) instead of multiple IPs

Common DNS Issues

Issue 1: Domain Not Resolving

Symptoms: Browser shows "This site can't be reached", DNS query returns no results
Diagnosis Commands 
# Check DNS resolution
dig vsbot.site
nslookup vsbot.site

# Check nameservers
dig NS vsbot.site +short

# Check specific record type
dig A vsbot.site +short
Solutions:
  • Verify nameservers updated at registrar
  • Wait for DNS propagation (up to 48 hours)
  • Verify DNS records exist in Route 53
  • Check DNS propagation globally: whatsmydns.net

Issue 2: Wrong IP Address Returned

Clear DNS Cache 
# Windows
ipconfig /flushdns

# Linux
sudo systemd-resolve --flush-caches

# Mac
sudo dscacheutil -flushcache

Issue 3: SSL Certificate Issues

Check and Renew Certificate 
# Check certificate expiration
echo | openssl s_client -connect vsbot.site:443 2>/dev/null | openssl x509 -noout -dates

# Renew certificate
sudo certbot renew
sudo systemctl reload nginx

# Force renewal
sudo certbot --nginx -d vsbot.site -d www.vsbot.site --force-renewal

# Check Certbot auto-renewal
sudo systemctl status certbot.timer
